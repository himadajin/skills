# Create Setting Reference

## 目的

これまでの会話に散らばっているシナリオ設定を整理し、作者本人が参照するための単一HTMLの設定資料集を作成する。

この資料集は、作品を魅力的に紹介するためのものではない。
会話中ですでに確定している事実を、検索・参照しやすい形に再構成するためのものである。
情報を充実させることよりも、既存情報を正確に整理することを優先する。

## 最重要原則

- HTMLのすべての記述は、会話中の確定した発言まで遡れなければならない。推測、補完、創作をせず、複数の発言を組み合わせてどの発言にもない新しい事実を作らない。
- 要約や統合は、情報を減らす方向にだけ行う。圧縮の過程で因果、動機、解釈を付け足さない。
- 作者の発言とアシスタント側の提案を区別し、作者が採用を明示していない案を確定設定として扱わない。
- 会話上の仮称を正式名称として扱わない。不明な名称や項目を、自然そうな内容で埋めない。
- 会話を発言順に要約せず、対象ごとに情報を統合する。同じ人物、関係、用語、出来事に関する記述は一つにまとめる。
- 一般論、助言、アシスタント側の提案、雑談は設定資料に含めない。

## 確定・未定・除外の判定

会話から抽出した情報は、次の3つに振り分ける。

1. **確定** — 作者が採用を明示した情報。HTMLに書く。
2. **未定** — 話題に上がったが、採用が明示された案がない項目。HTMLに「未定」と書く。候補案がある場合、候補はHTMLに書かず、チャット本文側に列挙する。
3. **除外** — 解釈が一意に定まらない情報、矛盾を含む情報。HTMLに含めず、チャット本文側に記載する。矛盾する発言の一方だけを作者が確定と明示している場合は、その側をHTMLに書き、矛盾はチャット本文側で確認する。どちらとも言えない場合は、矛盾しない粗い事実まで精度を下げてHTMLに書き、矛盾部分だけを除外する。

会話で一度も扱われていない項目は「未定」として追加しない。
存在しない情報を埋めるための欄を作らない。

正式名称が未定で、会話中の呼称だけがある場合は、正式名称を創作せず、必要に応じて次のように区別する。

```text
名称：未定
会話上の呼称：敵組織
```

## 出力

次の2つを出力する。

1. 確定設定だけを含む単一HTMLファイル
2. HTMLに含めなかった情報を説明するチャット本文(応答テキスト)

HTMLは、ファイルを保存できる環境では、指定がなければ作業ディレクトリに `setting-reference.html` として保存する。
Artifact などのプレビュー機能がある環境ではそれを用いる。

チャット本文には、該当するものがある場合だけ、次を簡潔に記載する。

- 検討中の案
- 矛盾または確認が必要な点
- 時系列に配置できなかった確定場面
- 解釈が一意に定まらずHTMLに含めなかった情報

曖昧な情報をHTMLへ無理に入れず、外側へ退避させることは、このスキルの重要な役割である。

## HTMLの構成

HTMLは次の4章で構成し、これ以外の章を追加しない。
情報が存在しない章は省略してよい。章間の必要な重複は許容する。
どの章にも属さない確定情報は、最も関連の深い既存項目の説明に含める。専用の項目を作らない。

1. **登場人物** — 人物ごとに、基本情報・概要・経歴・過去・目的・特徴を整理する。
2. **人間関係** — 人物ペアごとに、客観的な関係と双方向の認識を整理する。
3. **固有用語** — 人物以外の作品固有の名称や概念を、五十音順の辞書として整理する。
4. **時系列** — 作中世界で出来事が起きる順序を、確定している精度のまま場面単位で整理する。

タイトルは章にせず、文書上部に表示する。タイトルが未定の場合は「タイトル:未定」とする。

タイトルの直下にログラインを表示する。
ログラインは、作品の前提と主人公の行動を一続きに述べる短い要約であり、設定の羅列や網羅をしない。
作者が確定させたログラインがあればそれを転記する。
ない場合は、確定した事実だけで構成して生成する。因果や動機の接続は作者が明示した範囲でだけ使い、動機、テーマ、賭け金が未確定なら、補わず欠けたまま書く。

各章の項目定義・記入例・章別ルールは [references/chapters.md](references/chapters.md) に従う。

体裁は [references/format.md](references/format.md) の原則に従う。
デフォルトでは [assets/template.html](assets/template.html) をそのまま使い、プレースホルダーを内容で置き換える。
作品の雰囲気に合わせた装飾やデザインの調整をしない。

## 実行手順

1. 会話全体から、作品に関する記述を抽出する。
2. 各情報の確定・未定・除外を判定し、登場人物、人間関係、固有用語、時系列へ分類する。
3. 同一対象の別名や表記揺れを確認し、同一対象に関する確定情報を統合する。
4. references/chapters.md と references/format.md に従って、4章構成の単一HTMLを生成する。
5. references/chapters.md 末尾の最終確認チェックリストで確認する。
