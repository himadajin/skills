# 章別ルーブリック

HTML本体の4章それぞれの項目定義・記入例・章別ルール。
SKILL.md の最重要原則と「確定・未定・除外の判定」を前提とする。

## 1. 登場人物

登場人物は、人物ごとに整理する。
作中で意思を持って行動する存在は、AIや動物であっても登場人物として扱う。

人物名を見出しとし、次の5項目を使用する。

1. 基本情報
2. 概要
3. 経歴・過去
4. 目的
5. 特徴

### 基本情報

人物を識別するための客観的な情報を書く。

例:

- 年齢
- 性別
- 所属
- 役職
- 作中で明示された立場

固定の小項目をすべて埋める必要はない。
会話中で確定している情報だけを書く。

名前が未定で、会話上の識別名がある場合:

```text
名前:未定
会話上の呼称:主人公
```

「主人公」などの会話上の呼称を正式な人物名として扱わない。

### 概要

その人物が誰であるかを、確定情報だけで短くまとめる。

他の項目の長い再掲は行わない。

### 経歴・過去

現在の立場、目的、人間関係、重要な出来事に影響する過去を書く。

### 目的

シナリオ中で、その人物が何をしようとしているかを書く。

外から確認できる具体的な目的を優先する。

作者が明示していない深層心理、無意識の欲求、テーマ上の役割を推測しない。

### 特徴

確定した特徴を書く。
性格、身体的特徴、能力、制約などを必要に応じて含めてよい。

ただし、台詞や行動から性格や心理を推定しない。

### 登場人物章に含めないもの

- 人間関係の詳細
- 能力や制度の詳細なルール
- 出来事の順序

これらは該当する章へ記載する。

## 2. 人間関係

人間関係は、人物ごとではなく人物ペアごとに整理する。

各人物ペアについて、次の3項目を使用する。

1. 関係
2. 人物Aから人物B
3. 人物Bから人物A

### 関係

両者の間にある、客観的に確定した事実を書く。

例:

- 家族関係
- 所属上の関係
- 契約関係
- 師弟関係
- 協力関係
- 対立関係
- 両者の間で起きた重要な出来事

「親友」「宿敵」などの評価的な表現は、作者が明示した場合だけ使用する。

### 人物Aから人物B

人物Aが人物Bに対して持つ、明示された認識、感情、目的、態度を書く。

### 人物Bから人物A

人物Bが人物Aに対して持つ、明示された認識、感情、目的、態度を書く。

### 人間関係章のルール

- 少なくとも一つの確定した関係情報がある人物ペアだけを載せる。
- 一方向の情報しかない場合、反対方向を推測で埋めない。
- 出来事の詳細な順序は時系列章へ置く。

## 3. 固有用語

固有用語は、人物以外の作品固有の名称や概念を引くための辞書として扱う。
組織、場所、事件、制度、能力、道具、種族、作品独自の概念などを必要に応じて含める。

各項目は次の形式とする。

```text
用語名

その用語について確定している説明。
```

名称は見出しとして表示する。

名称が未定の場合は、会話上の呼称を見出しにし、本文の冒頭で名称が未定であることを明示する。

```text
カンパニー

名称:未定(「カンパニー」は会話上の呼称)
その組織について確定している説明。
```

### 固有用語章のルール

- 一般語は、作品独自の意味や定義がある場合だけ載せる。
- 会話中の一時的な説明用呼称を正式名称として扱わない。
- 説明には確定情報だけを書く。
- 名称の五十音順に並べる。英数字の名称は五十音の後にA–Z順でまとめ、「名称:未定」の項目はさらにその後にまとめる。

## 4. 時系列

時系列章は、一意の年表を無理に作るためのものではない。

確定している時間関係だけを保存する。

扱うのは、作中世界で出来事が起きる順序である。
シナリオ上の提示順、回想の配置、編集上の並び順は扱わない。

### 基本単位

時系列は場面単位で整理する。

各場面は次の3要素を持つ。

1. 時点
2. 場面
3. 場面内の流れ

### 時点

会話から確定している精度をそのまま保持する。

例:

```text
2032年4月12日 14時ごろ
物語開始の3日前
「蓮とミナの再会」の後
物語開始以前
```

日時の精度を勝手に上げない。

「物語開始以前」としか分からないものを、「数年前」などと補わない。

### 場面

場面を識別できる、事実ベースの短い説明を書く。

正式な場面名がある場合はそれを使用する。

正式名称がない場合は、次のような説明的な見出しを使う。

```text
蓮が病院を訪れる場面
```

創作的な場面タイトルを付けない。

避ける例:

```text
再会の朝
運命の邂逅
```

### 場面内の流れ

場面内で確定している出来事と順序を書く。

順序が完全に確定している場合は番号付きリストを使う。

```text
1. 蓮が病院に到着する。
2. 黒田と会う。
3. ミナと再会する。
```

一部の順序だけが確定している場合は、確定部分だけを書く。

不明な出来事を自然な順序へ並べない。
因果関係から時間関係を推定しない。

### 場面間の順序

異なる場面の前後関係が確定している場合だけ、その順序を表現する。

同じ時期区分に複数の場面があり、前後関係が確定していない場合、掲載順に意味を持たせない。

時系列章の冒頭に、次の注意を記載する。

```text
同じ時期区分内の掲載順は、明示されていない限り前後関係を表さない。
```

### 時系列に配置できない場面

場面の存在は確定しているが、時系列上の位置が定まらない場合、HTMLへ無理に配置しない。

HTMLの外側(チャット本文)に次のように記載する。

```text
時系列に配置できなかった確定場面
- 黒田が白夜事件の資料を入手する場面
  時系列上の位置が未定。
```

## 最終確認チェックリスト

HTMLを出力する前に、次を確認する。

- 会話に存在しない名称や説明を創作していないか。
- アシスタント側の提案を確定設定として採用していないか。
- 性格や心理を台詞や行動から推定していないか。
- 日時や順序の精度を勝手に上げていないか。
- 創作的な場面名や仮タイトルを付けていないか。
