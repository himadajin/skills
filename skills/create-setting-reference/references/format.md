# 体裁

設定資料集HTMLの体裁の規範。

体裁の目的は、どの作品にも馴染む無色透明な資料にすることである。
作品の雰囲気に合わせた配色、装飾、デザインの調整をしない。

## 原則

- 単一HTMLファイルとする。外部CSS、外部JavaScript、CDN、外部フォントを読み込まない。JavaScriptを使わない。
- ライトモード固定。色はグレースケールのみ。アイコン、絵文字、意味を運ばない装飾を使わない。線や図形は、順序や構造などの意味を運ぶ場合にだけ使う。
- グレースケール内でも、濃度や背景の差は意味の差があるところにだけ使う。
- 本文はシステムフォントのゴシック体で、読みやすい本文幅に収める。
- 印刷しても読める体裁とする。

## テンプレート

デフォルトでは [assets/template.html](../assets/template.html) をそのまま使い、プレースホルダーを内容で置き換える。

テンプレートを参照できない環境や、内容上の必要がある場合は、上記の原則を守る範囲で体裁を構成してよい。

## 時系列章の表示

時系列章は、章の左に1本の縦レールを通し、各時点(h3)の位置にマーカーを置いた縦型タイムラインとして表示する。

- レール上のマーカーの並びで表すのは出来事の前後関係だけであり、縦方向の間隔や位置に時間の長さの意味を持たせない。
- 同じ時点に属する複数の場面は、1つのマーカーを共有する。マーカーの共有は、場面間の順序が未確定であることを表す。
- 時間の長さや間隔を主張する表現(目盛り付きの軸、横型タイムライン、ガント風の表)を使わない。確定している精度以上の情報を図の側で作らない。

## マークアップの規約

- 見出し階層は、h1=文書タイトル、h2=章、h3=項目(人物名、人物ペア、用語名、時点)、h4=項目内の小見出し(基本情報、関係、場面など)とする。
- ログラインはh1直下に段落として置き、章にしない。
- 章の `id` は `characters` / `relationships` / `terms` / `timeline` に固定する。
- 項目の `id` は内容が識別できるものを付ける。日本語のままでよい(例: `id="蓮"`)。
- 目次には章と項目(h3相当)までを載せる。
- キーと値の組(基本情報など)は `dl` を使う。順序が確定した流れは `ol`、順不同の列挙は `ul` を使う。
- 情報が存在しない章・項目は、目次からも本文からも省略する。
