# テンプレート方針

技術文書の執筆・改稿を始める前に、どのテンプレートを使うかを確定する。

## 目的

情報提供順序と章立てを初期段階で固定し、後工程での大規模な手戻りを防ぐ。
文書の目的に適したテンプレートを選ぶことで、読者の理解を助け、書き手と読み手の認識のズレを防ぐ。

## テンプレートモード

- `inherit`: 元資料や既存文書の体裁を継承する。
- `user_template`: ユーザーが与えたテンプレートを適用する。
- `skill_default`: 上記がない場合に、文書の目的に応じたスキル標準テンプレートを使う。

## 優先順位

1. ユーザーの明示指示
2. 元資料・既存文書の制約
3. スキル標準テンプレート（目的別）

指示が競合する場合は、優先順位の高いものを採用し、採用理由を明示する。

## 事前確認チェックリスト

着手前に次を明確化する。

- `template_mode`: `inherit` / `user_template` / `skill_default`
- `template_type`: `skill_default` の場合、テンプレートカタログから選択したテンプレート ID
- `must_keep`: 維持必須の章、見出し、順序、表記規則
- `can_change`: 再編可能な範囲

不足情報がある場合は、勝手に最適化せず、仮定を明示して進める。

## テンプレート選択フロー

`skill_default` モードの場合、以下の手順でテンプレートを選択する。

1. 文書の主目的を特定する。「この文書は読者に何をさせたいか、何を伝えたいか」を一文で表す。
2. テンプレートカタログの選択基準と照合し、最も適合するテンプレートを選ぶ。
3. 選択したテンプレートをユーザーに提示し、確認を得てから執筆を開始する。

複数のテンプレートが該当しうる場合は、文書の主目的に最も近いものを選び、選択理由を明示する。
どのテンプレートにも該当しない場合は、最も近いテンプレートをベースに節構成を調整し、調整内容を明示する。

## テンプレートカタログ

`skill_default` モードで使用するテンプレートの一覧を以下に示す。各テンプレートの詳細は `templates/` ディレクトリの対応ファイルを参照すること。

| ID | 名称 | 主目的 | テンプレートファイル |
|----|------|--------|---------------------|
| `explanation` | 技術説明書 | 対象の仕組み・概念を読者に理解させる | `templates/explanation.md` |
| `investigation-report` | 調査報告書 | 調査の結果と考察を報告する | `templates/investigation-report.md` |
| `design-doc` | 設計文書 | 設計判断とその根拠を記録する | `templates/design-doc.md` |
| `howto` | 手順書 | 手順を正確に伝え、読者が再現できるようにする | `templates/howto.md` |
| `incident-report` | 障害報告書 | 障害の経緯・原因・再発防止策を記録する | `templates/incident-report.md` |

### 選択の判断基準

迷った場合は以下の問いで切り分ける。

- 読者に仕組みを理解させたい → `explanation`
- 調べた結果を報告したい → `investigation-report`
- 設計の判断と根拠を残したい → `design-doc`
- 読者に作業を実行させたい → `howto`
- 障害の記録と再発防止策を共有したい → `incident-report`
