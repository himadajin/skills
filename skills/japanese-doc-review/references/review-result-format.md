# レビュー結果の形式

レビュー結果は、この固定形式だけで構成する。
固定形式の外側に、本文、修正版、要約、補足説明を置かない。

## 固定テンプレート

```markdown
対象範囲: 全文

規定事項:
- なし

確認事項:
- なし

## 構成・論理
今回のレビュー範囲: 含む
理由: 指定なしのため、最も上流の観点を優先。

指摘:
- なし

## 文意・文法
今回のレビュー範囲: 含まない
理由: 構成・論理のレビューを優先。

指摘:
- 今回はレビュー対象外。

## 体裁・表現
今回のレビュー範囲: 含まない
理由: 構成・論理のレビューを優先。

指摘:
- 今回はレビュー対象外。
```

## 置換規則

- 固定テンプレートの見出しとフィールド名は変えない。
- `対象範囲` には、今回レビューした実在本文の範囲を書く。
- `規定事項` は常に出す。該当する規定事項がない場合は `- なし` と書く。
- `確認事項` は常に出す。確認事項がない場合は `- なし` と書く。
- 観点見出しは `構成・論理`、`文意・文法`、`体裁・表現` の3つを常にこの順序で出す。
- `今回のレビュー範囲` の値は `含む` または `含まない` にする。
- `理由` には、レビュー範囲を含むかどうかの判断理由を短く書く。
- レビュー範囲に含む観点で指摘がない場合、`指摘` は `- なし` と書く。
- レビュー範囲に含まない観点では、`指摘` は `- 今回はレビュー対象外。` と書く。

`- なし` は、その観点で今回のレビュー範囲に含め、具体的な対象と根拠を示せる指摘が見つからなかったことを意味する。
他の観点でも問題がないことを意味しない。

## 指摘ブロック

指摘がある場合は、該当する観点の `指摘:` に次の形式で書く。

```markdown
指摘:

**指摘 1**
重要度: 高
対象: skills/example.md:12 の文: 「...」

問題:
...

根拠:
...

修正方針:
...
```

`対象` には、位置と引用を含める。
`修正方針` には、修正後の本文ではなく、修正の判断基準または作業方針を書く。
