# レビュー結果の形式

レビュー結果は、この固定形式だけで構成する。
固定形式の外側に、本文、修正版、要約、補足説明を置かない。
レビュー結果ファイルに書き出す場合だけ、固定形式の前に `次の指摘ID: ISSUE-001` の形式で次に新規指摘へ割り当てるIDを書く。

## 固定テンプレート

```markdown
対象範囲: 全文

規定事項:
- なし

確認事項:
- なし

## 構成・論理
今回のレビュー範囲: 含む
理由: 指定なしのため、最も上流の観点を優先。

指摘:
- なし

## 文意・文法
今回のレビュー範囲: 含まない
理由: 構成・論理のレビューを優先。

指摘:
- 今回はレビュー対象外。

## 体裁・表現
今回のレビュー範囲: 含まない
理由: 構成・論理のレビューを優先。

指摘:
- 今回はレビュー対象外。
```

## 置換規則

- 固定テンプレートの見出しとフィールド名は変えない。
- `対象範囲` には、今回レビューした実在本文の範囲を書く。
- `規定事項` は常に出す。該当する `規定事項` がない場合は `- なし` と書く。
- `確認事項` は常に出す。確認事項がない場合は `- なし` と書く。
- 観点見出しは `構成・論理`、`文意・文法`、`体裁・表現` の3つを常にこの順序で出す。
- `今回のレビュー範囲` の値は `含む` または `含まない` にする。
- `理由` には、レビュー範囲を含むかどうかの判断理由を短く書く。
- レビュー範囲に含む観点で指摘がない場合、`指摘` は `- なし` と書く。
- レビュー範囲に含まない観点では、`指摘` は `- 今回はレビュー対象外。` と書く。

`- なし` は、その観点で今回のレビュー範囲に含め、具体的な対象と根拠を示せる指摘が見つからなかったことを意味する。
他の観点でも問題がないことを意味しない。

## 指摘ブロック

指摘がある場合は、該当する観点の `指摘:` に次の形式で書く。

```markdown
指摘:

**ISSUE-001**
重要度: 高
対象: `skills/example.md` の「## 使う場面」内の文「...」

問題:
...

根拠:
...

修正方針:
...
```

指摘見出しには、`ISSUE-001` のような3桁0埋めのASCII指摘IDを書く。
指摘が複数ある場合は、`ISSUE-002`、`ISSUE-003` のように続ける。
レビュー結果ファイルを使わない場合は、`ISSUE-001` から順に付ける。
レビュー結果ファイルを使う場合は、既存指摘のIDを維持し、新規指摘にはファイル先頭の `次の指摘ID` の値を使う。

`対象` には、位置と引用を含める。
ファイルをレビューする場合は、ファイルパス、見出し、節名、段落、箇条書き位置、短い引用を優先して示す。
行番号は、指摘位置を特定するために必要な場合だけ使う。
`修正方針` には、修正後の本文ではなく、修正の判断基準または作業方針を書く。

再レビューで既存指摘が残る場合も、指摘文は今回レビューした本文だけを根拠に初見の指摘として読めるように書く。
`前回`、`まだ`、`引き続き`、`未修正`、`同様` のような過去のレビュー結果に依存する表現を使わない。
