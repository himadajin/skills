# TYPOの方針

`TYPO` では、明らかで修正が一択の誤字、脱字、余字、固有名詞や用語の誤記だけを扱う。
修正者が意味判断をせずに、単一置換として直せるものに限る。

## 指摘対象

`TYPO` として扱うもの:

- 文字の抜け、余分な文字、明らかな変換ミス。
- 同じ文書内に正式表記があり、誤った表記と正しい表記が一対一で決まる固有名詞や用語の誤記。
- 本文中の表記ルールから、修正後の文字列が一意に決まる誤記。

`TYPO` として扱わないもの:

- 修正候補が複数あるもの。
- 修正によって文意が変わるもの。
- 外部検索、辞書、専門知識がないと正誤を判断できないもの。
- 表記統一、文体、冗長表現の問題。
- スペースの有無だけの問題。

直し方が一択でないもの、または修正によって文意が変わるものは `GRAMMAR` で扱う。
同じ概念の表記統一は `STYLE` で扱う。

## スペース有無

スペースの有無だけを理由に `TYPO` の指摘を出さない。
ただし、コード、コマンド、識別子、固有名詞などでスペースの有無が意味を変える場合は対象外にしない。

## 指摘形式

`TYPO` の指摘には、重要度、問題、根拠を書かない。
`対象` には、誤っている原文を必ずバッククォートで引用する。
同じ誤記が複数箇所にある場合は1つの指摘にまとめ、全出現箇所を示す。

`修正方針` は、必ず次の単一置換形式で書く。

```markdown
`誤` を `正` に修正する。
```

この形式で書けないものは `TYPO` として扱わない。
