# 日本語文章ルール

日本語の文章を執筆、編集、チェックする際に、以下の 3 つのレビュー項目集に従って作業する。ルールは「レビュー項目」として位置付ける。執筆・編集時はレビュー項目を参照しながら書き、完成後に同じ項目でレビューして通す。

- `references/01-structure.md`: 構成レベル。章構成・情報の導入順序・スコープ・整合性など、文書全体の骨組みに関する項目。
- `references/02-writing.md`: 記述レベル。文法・言葉遣いの方針。執筆中にも意識しやすいよう方針だけを記す。
- `references/03-formatting.md`: 体裁レベル。Markdown 体裁と、具体的な禁止パターンの一覧。機械的に照合する。

## ルール

- 上記 3 ファイルに記載されたレビュー項目に従って作業を行う。
- 敬体と常体は作業対象の文章に合わせる。ユーザーの指示がある場合はそれに従う。どちらも判断できない場合は常体にする。

## ワークフロー

1. 作業の種類（執筆・編集・チェック）と要件を確認する。
2. 以下の 3 段階レビューを順に適用する。
   1. 構成レビュー: `references/01-structure.md` の項目で原稿の骨組みをレビューし、必要な修正を行う。
   2. 記述レビュー: `references/02-writing.md` の項目で文法・言葉遣いをレビューし、必要な修正を行う。
   3. 体裁レビュー: `references/03-formatting.md` の項目で Markdown 体裁と禁止パターンを照合し、必要な修正を行う。
3. 既定では 3 段階を連続して実行し、各段階で何を変更したかを差分として報告する。段階の境界で自動停止はしない。
4. 作業の完了前に、3 ファイルすべてのレビュー項目に反する箇所がないかを必ず確認する。

作業種別（執筆・編集・チェック）にかかわらず 3 段階レビューを基本とする。指示が曖昧な場合も 3 段階を既定とする。

## 柔軟性

ユーザーが以下のいずれかを指示した場合はそれに従う。

- 特定段階だけ適用する（例: 体裁だけチェック）。
- 段階の境目でいったん停止し、確認や質問を挟む。
- 段階の順を入れ替える、または特定段階をスキップする。

## 出力

ユーザーが出力形式を指定した場合は指示に従い、指定がない場合は以下の方針に従う。

- 執筆: 完成した本文を返す。
- 編集: 修正文を返す。主要な修正点の説明が有用な場合だけ添える。
- チェック: 問題点を返す。修正案が有用なら併記する。
