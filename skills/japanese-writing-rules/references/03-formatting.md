# 体裁のレビュー項目

書き終わった文章に対して機械的に照合する項目を示す。Markdown の体裁と、`02-writing.md` の方針に対応する具体的な禁止パターンの一覧を置く。

## Markdown 体裁

Markdown 記法の太字(`**`)を使用しないこと。
太字は読み手の環境によって表示が異なるため、認識のズレにつながる。
また、太字を使わないと情報の重要度が伝わりにくくなる文章は書くべきではない。
情報の重要度は文章で表現すること。

## 冗長語句の候補

以下の語句が含まれている場合、削除して文意が通るかを確認する。
文意が通るなら削除する。
文意が通らない場合は、語句を残すのではなく、文の構造を見直して語句なしで意味が通る文に書き直す。
ただし「Aとしては正しいがBとしては誤りである」のように明示的な対比構文で使う場合は残してよい。

- 「としては」
- 「においては」
- 「に対して」
- 「についての」
- 「という」

## 範囲曖昧化の候補

以下の語で範囲を曖昧にしないこと。

- 「など」
- 「等」

「AやBなど」と書いた場合、読み手は「など」に何が含まれるかを判断できない。
列挙が網羅的であれば「など」を外す。
網羅できない場合は、列挙した項目の上位概念で範囲を示すか、条件を明記して範囲を限定する。

## 曖昧な推量表現の候補

以下の推量表現は避ける。

- 「思われる」
- 「考えられる」
- 「可能性がある」

事実であれば断定し、不確かであれば根拠や条件を明示する。

## 体言止めの検出

文末が名詞で終わっていないかを機械的に確認する。
該当する文は、動詞や助動詞で文を終える形に書き直す。
ただし、箇条書きの項目名やタイトルで名詞を用いることはこの規則の対象外とする。
