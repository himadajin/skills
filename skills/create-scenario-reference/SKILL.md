# Create Scenario Reference

## 目的

これまでの会話に散らばっているシナリオ設定を整理し、作者本人が参照するための単一HTMLの設定資料集を作成する。

この資料集は、作品を魅力的に紹介するためのものではない。  
会話中ですでに確定している事実を、検索・参照しやすい形に再構成するためのものである。

情報を充実させることよりも、既存情報を正確に整理することを優先する。

## 最重要原則

- 会話から確実に読み取れる事実だけをHTMLに書く。
- 情報を推測、補完、創作しない。
- 複数の解釈が可能な情報はHTMLに含めない。
- 検討中の案、矛盾、確認事項、未解決事項はHTMLの外側に書く。
- 会話上の仮称を正式名称として扱わない。
- 不明な名称や項目を、自然そうな内容で埋めない。
- 作者が採用したと明示していない案を確定設定として扱わない。
- 会話を発言順に要約せず、対象ごとに情報を統合する。
- 同じ人物、関係、用語、出来事に関する記述は一つにまとめる。
- 一般論、助言、ChatGPT側の提案、雑談は設定資料に含めない。

## 未定の扱い

話題に上がり、未決であることが明確な項目は「未定」と書く。

例:

```text
タイトル：未定
```

ただし、会話で一度も扱われていない項目まで「未定」として追加しない。  
存在しない情報を埋めるための欄を作らない。

正式名称が未定で、会話中の呼称だけがある場合は、正式名称を創作せず、必要に応じて次のように区別する。

```text
名称：未定
会話上の呼称：敵組織
```

## 出力

次の2つを出力する。

1. 単一HTMLファイル
2. HTMLに含めなかった情報を説明するチャット本文

HTMLは確定設定だけを含む。

チャット本文には、該当するものがある場合だけ、次を簡潔に記載する。

- 検討中の案
- 矛盾または確認が必要な点
- 時系列に配置できなかった確定場面
- 解釈が一意に定まらずHTMLに含めなかった情報

曖昧な情報をHTMLへ無理に入れず、外側へ退避させることは、このスキルの重要な役割である。

# HTMLの構成

HTMLは次の4章で構成する。

1. 登場人物
2. 人間関係
3. 固有用語
4. 時系列

タイトルは章にせず、文書上部に表示する。

```text
タイトル：確定タイトル
```

タイトルが決まっていない場合:

```text
タイトル：未定
```

上記4章以外の章を追加しない。

情報が存在しない章は省略してよい。  
空欄を埋めるために内容を生成してはならない。

# 1. 登場人物

登場人物は、人物ごとに整理する。

人物名を見出しとし、次の5項目を使用する。

1. 基本情報
2. 概要
3. 経歴・過去
4. 目的
5. 特徴

## 基本情報

人物を識別するための客観的な情報を書く。

例:

- 年齢
- 性別
- 所属
- 役職
- 作中で明示された立場

固定の小項目をすべて埋める必要はない。  
会話中で確定している情報だけを書く。

名前が未定で、会話上の識別名がある場合:

```text
名前：未定
会話上の呼称：主人公
```

「主人公」などの会話上の呼称を正式な人物名として扱わない。

## 概要

その人物が誰であるかを、確定情報だけで短くまとめる。

一文から三文程度を目安とする。

人物像を魅力的に見せるための説明や、他の項目の長い再掲は行わない。

## 経歴・過去

現在の立場、目的、人間関係、重要な出来事に影響する過去を書く。

人物の生涯を網羅しない。

出来事の時間的位置は、必要に応じて時系列章にも記載してよい。

## 目的

シナリオ中で、その人物が何をしようとしているかを書く。

外から確認できる具体的な目的を優先する。

作者が明示していない深層心理、無意識の欲求、テーマ上の役割を推測しない。

## 特徴

次のいずれかに該当する、確定した特徴を書く。

- 人物の識別に必要
- 行動や選択に影響する
- 他の設定との整合性確認に使える

性格、身体的特徴、能力、制約などを必要に応じて含めてよい。

ただし、台詞や行動から性格や心理を推定しない。  
外見を網羅するキャラクターデザイン資料にはしない。

## 登場人物章に含めないもの

- 人間関係の詳細
- 能力や制度の詳細なルール
- 出来事の順序
- 物語上の役割についての分析
- 作者が明示していない性格や心理

これらは、必要に応じて他章へ記載するか、HTMLの外側へ出す。

# 2. 人間関係

人間関係は、人物ごとではなく人物ペアごとに整理する。

各人物ペアについて、次の3項目を使用する。

1. 関係
2. 人物Aから人物B
3. 人物Bから人物A

## 関係

両者の間にある、客観的に確定した事実を書く。

例:

- 家族関係
- 所属上の関係
- 契約関係
- 師弟関係
- 協力関係
- 対立関係
- 両者の間で起きた重要な出来事

「親友」「宿敵」などの表現は、作者が明示した場合だけ使用する。

## 人物Aから人物B

人物Aが人物Bに対して持つ、明示された認識、感情、目的、態度を書く。

## 人物Bから人物A

人物Bが人物Aに対して持つ、明示された認識、感情、目的、態度を書く。

## 人間関係章のルール

- 全人物の総当たりを作らない。
- 少なくとも一つの確定した関係情報がある人物ペアだけを載せる。
- 単に同じ場所にいた、会話したというだけでは項目を作らない。
- 関係種別の固定ラベルを付けない。
- 関係図を作らない。
- 一方向の情報しかない場合、反対方向を推測で埋めない。
- 関係の変化は、必要な場合だけ時点を添えて簡潔に書く。
- 出来事の詳細な順序は時系列章へ置く。

形式:

```text
人物Aと人物B

関係
客観的に確定している関係や出来事。

人物Aから人物B
人物Aが人物Bに対して持つ、明示された認識・感情・目的。

人物Bから人物A
人物Bが人物Aに対して持つ、明示された認識・感情・目的。
```

# 3. 固有用語

固有用語は、人物以外の作品固有の名称や概念を引くための辞書として扱う。

各項目は次の形式とする。

```text
用語名

その用語について確定している説明。
```

名称は見出しとして表示する。  
種別、関連人物、初出、目的などの固定項目は設けない。

## 対象

必要に応じて次を含める。

- 組織
- 集団
- 場所
- 事件
- 計画
- 制度
- 技術
- 魔法
- 能力
- 道具
- 病気
- 種族
- 役職
- 作品独自の概念

## 固有用語章のルール

- 一般語は、作品独自の意味や定義がある場合だけ載せる。
- 会話中の一時的な説明用呼称を正式名称として扱わない。
- 名称が未定なら「未定」と明示する。
- 説明には確定情報だけを書く。
- 用語を分類しない。
- 原則として名称の五十音順に並べる。
- 英数字の名称の位置は文書内で統一する。
- 「名称：未定」の項目は末尾にまとめる。
- 登場人物や時系列との必要な重複は許容する。

情報が少ない場合は短い文章で書き、情報が多い場合は箇条書きを使用してよい。

# 4. 時系列

時系列章は、一意の年表を無理に作るためのものではない。

確定している時間関係だけを保存する。

扱うのは、作中世界で出来事が起きる順序である。  
シナリオ上の提示順、回想の配置、編集上の並び順は扱わない。

## 基本単位

時系列は場面単位で整理する。

各場面は次の3要素を持つ。

1. 時点
2. 場面
3. 場面内の流れ

HTML上では、時点を上位見出し、場面を下位見出しとしてよい。

## 時点

会話から確定している精度をそのまま保持する。

例:

```text
2032年4月12日 14時ごろ
```

```text
物語開始の3日前
```

```text
「蓮とミナの再会」の後
```

```text
物語開始以前
```

日時の精度を勝手に上げない。

「物語開始以前」としか分からないものを、「数年前」などと補わない。

## 場面

場面を識別できる、事実ベースの短い説明を書く。

正式な場面名がある場合はそれを使用する。

正式名称がない場合は、次のような説明的な見出しを使う。

```text
蓮が病院を訪れる場面
```

創作的な場面タイトルを付けない。

避ける例:

```text
再会の朝
```

```text
運命の邂逅
```

## 場面内の流れ

場面内で確定している出来事と順序を書く。

順序が完全に確定している場合は番号付きリストを使う。

```text
1. 蓮が病院に到着する。
2. 黒田と会う。
3. ミナと再会する。
```

一部の順序だけが確定している場合は、確定部分だけを書く。

不明な出来事を自然な順序へ並べない。

## 場面間の順序

異なる場面の前後関係が確定している場合だけ、その順序を表現する。

同じ時期区分に複数の場面があり、前後関係が確定していない場合、掲載順に意味を持たせない。

時系列章の冒頭に、次の注意を記載する。

```text
同じ時期区分内の掲載順は、明示されていない限り前後関係を表さない。
```

## 時系列に配置できない場面

場面の存在は確定しているが、時系列上の位置が定まらない場合、HTMLへ無理に配置しない。

HTMLの外側に次のように記載する。

```text
時系列に配置できなかった確定場面
- 黒田が白夜事件の資料を入手する場面
  時系列上の位置が未定。
```

## 時系列章のルール

- 作中世界での時間順を扱う。
- シナリオ上の提示順は扱わない。
- 場面単位で整理する。
- 日時の精度は、会話で確定している範囲を維持する。
- 場面内の順序も、確定している部分だけを書く。
- 一意に決まらない場面同士を無理に並べない。
- 因果関係から時間関係を推定しない。
- 同じ時期区分内の掲載順には、原則として意味を持たせない。
- 時系列上の位置が定まらない場面はHTMLの外側へ出す。
- 正式な場面名がなければ、事実ベースの説明を用いる。
- 創作的な場面名や日時を補わない。

# HTMLの体裁

HTMLは単一ファイルとする。

- CSSは`<style>`内に埋め込む。
- 外部CSS、外部JavaScript、CDN、外部フォントを使わない。
- JavaScriptは原則として使わない。
- 文書上部にタイトルを表示する。
- 目次を付ける。
- 各章と各項目にアンカーを付ける。
- 長文を読みやすい本文幅にする。
- スマートフォンでも読めるようにする。
- 印刷用CSSを用意する。
- 装飾は抑え、資料としての可読性を優先する。
- 色による複雑な状態分類を行わない。
- HTML内に検討中、矛盾、要確認などの状態表示を作らない。

# 実行手順

1. 会話全体から、作品に関する記述を抽出する。
2. 作者の発言とChatGPT側の提案を区別する。
3. 採用が明示された情報と、単なる候補を区別する。
4. 登場人物、人間関係、固有用語、時系列へ分類する。
5. 同一対象の別名や表記揺れを確認する。
6. 同一対象に関する確定情報を統合する。
7. 複数解釈が可能な情報をHTMLから除外する。
8. 矛盾や未決事項をHTMLの外側へ退避する。
9. 4章構成の単一HTMLを生成する。
10. 最終確認を行う。

# 最終確認

HTMLを出力する前に、次を確認する。

- 会話に存在しない名称を作っていないか。
- タイトルが未定なのに仮タイトルを付けていないか。
- ChatGPT側の提案を確定設定として採用していないか。
- 人物の性格や心理を行動から推定していないか。
- 人間関係の片側の情報から反対側を推定していないか。
- 一般語を不要に固有用語へ登録していないか。
- 場面間の順序を自然さや因果関係から補っていないか。
- 日時の精度を勝手に上げていないか。
- 創作的な場面名を付けていないか。
- 検討中の案、矛盾、確認事項がHTMLへ混入していないか。
- HTMLが登場人物、人間関係、固有用語、時系列の4章だけで構成されているか。
- 情報量を増やすための説明を追加していないか。
